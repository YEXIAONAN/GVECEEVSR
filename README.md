# 广西职教高考志愿模拟推荐

一个面向广西职教高考考生的前端静态志愿辅助模拟工具。  
项目当前聚焦电子信息大类与对口技能类录取方式，输出“学校 + 专业”粒度的分组推荐结果。

## 项目简介

本项目通过本地静态数据与透明规则，为考生提供志愿模拟参考。  
工具重点是“可解释、可维护、可扩展”，避免“黑盒式”结论输出。

## 项目定位

- 广西职教高考志愿辅助模拟工具。
- 用于辅助理解学校与专业的历史参考信息和风险分层。
- 项目目标是工具化实践，不是宣传页，不是后台系统。

## 当前支持范围

- 专业大类：电子信息大类
- 录取方式：对口技能类录取方式
- 推荐粒度：学校 + 专业
- 数据形态：本地静态数据（无后端、无数据库、无账号体系）

## 技术栈

- Vue 3
- Vite
- Vue Router
- JavaScript（无强制 TypeScript）
- 原生 CSS（无 UI 组件库）

## 本地开发方式

```bash
npm install
npm run dev
```

默认本地地址由 Vite 输出（通常为 `http://localhost:5173`）。

## 构建与部署方式

### 本地构建与预览

```bash
npm run build
npm run preview
```

### GitHub Pages 部署

仓库已提供工作流：`.github/workflows/deploy-gh-pages.yml`。  
`main` 分支更新后，流程将自动安装依赖、构建并发布 `dist/` 到 GitHub Pages。

项目在 `vite.config.js` 中支持 GitHub Pages `base` 路径：

- 生产默认：`/GVECEEVSR/`
- CI 可通过 `VITE_BASE_PATH` 覆盖为 `/${repoName}/`

## 数据说明

- 数据文件位于 `src/data/`，由人工维护。
- 核心字段覆盖：学校、专业、大类、录取方式、参考分数年份、最低分、计划数、来源、备注、数据状态。
- 缺失数据保持 `null`，页面明确展示“暂无公开数据”。
- 数据可能存在未补全或待更新项，更新记录以仓库提交为准。

## 推荐逻辑说明

- 推荐基于“用户分数 - 专业去年最低分”的分差规则。
- 分组策略：
  - `冲`：分差 0~8
  - `稳`：分差 9~20
  - `保`：分差 > 20
- 分差 < 0 的高风险项默认不作为主推荐展示。
- 无最低分的专业不参与常规分差计算，进入“仅计划参考”分组。
- 不做“同校只保留一个专业”的去重，严格按“学校 + 专业”逐条展示。

## 排名估算说明

- 排名模块为辅助信息，不是核心推荐依据。
- 采用预设规则做宽区间估算与竞争力等级表达。
- 输出示例为“参考位次（低可信）”，不输出精确名次承诺。
- 排名估算为低可信规则估算，不代表官方一分一档。
